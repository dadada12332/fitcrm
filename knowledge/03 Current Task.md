# Current Task

Исправить по порядку критичные и частые проблемы, найденные во время реальных смен владельца,
ресепшена и тренера в CRM.

## Готово

- Гранулярные permissions продублированы в RLS для клиентов, абонементов, оплат, посещений,
  расписания, склада, сотрудников, ролей и чувствительных служебных данных.
- DB-права пересекаются с доступами тарифа; cross-club внешние ссылки блокируются триггерами.
- Зарплаты, история выплат, долги/балансы клиентов и закупочные цены недоступны через Data API
  без специализированной server-side проверки.
- Финансы, зарплаты, роли и настройки больше не попадают в UI/RSC payload сотрудникам без прав.
- Устранены двойные активные абонементы; добавлен уникальный DB-инвариант и repair старых дублей.
- Заморозка считает остаток без отрицательных значений, хранит начало и продлевает срок корректно.
- Оплата без нового абонемента может погашать долг; комментарий сохраняется в истории.
- Время посещений нормализовано для `Asia/Tashkent`, созданные занятия без зала видны в календаре.
- Исправлены stale results глобального поиска, очистка клиентских фильтров и зависающий drawer check-in.
- Кнопки и вкладки скрываются по permissions, валидация форм стала понятнее.
- Миграция `20260724090925_granular_permissions_rls` применена в production Supabase.
- Дополнительные hardening-миграции `20260724131000`–`20260724145500` применены и проверены.
- Токены приглашений закрыты от Data API; роль приглашения ограничена правами приглашающего.
- Цена абонемента сверяется на сервере, Telegram-ссылка строится по scoped payment ID.
- Check-in атомарно проверяет клиента и абонемент, создаёт визит и списывает посещение одной RPC.
- TypeScript, production build и smoke основных CRM-маршрутов проходят без browser errors.
