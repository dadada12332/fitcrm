# Привести блок Внимание в отчётах к дизайн-системе

## Goal

Сделать вкладку «Внимание» в отчётах визуально единой с CRM и пригодной для ежедневной работы на desktop и mobile.

## Reason

Старый блок состоял из равнозначных прыгающих карточек, использовал сырые цвета и не отделял реальные задачи от нейтральной динамики посещаемости.

## Requirements

- Сгруппировать задачи в одну рабочую поверхность с понятным приоритетом и переходами.
- Отделить посещаемость от задач, сохранив переход в соответствующий отчёт.
- Использовать только дизайн-токены и обеспечить мобильную адаптацию.
- Исправить счётчик вкладки, чтобы он учитывал все реальные задачи.

## Acceptance criteria

- [x] Срочные, предупреждающие и нормальные состояния различимы без визуального шума.
- [x] Каждая активная строка ведёт в правильный раздел отчётов.
- [x] На мобильном экране нет горизонтального overflow и обрезанного текста.
- [x] Счётчик «Внимание» включает продления, зону риска и долги.
- [x] TypeScript, lint, tests и production build проходят.

## Files and data

- Files: `src/components/app/ReportsClient.tsx`.
- Tables/RPC: существующая агрегация `getReportsAlerts`; миграций нет.

## Must not break

Tenant isolation, auth, production data и существующие пользовательские сценарии.

## Changes

Четыре разрозненные карточки заменены единой рабочей поверхностью. Продления, зона риска и долги показаны приоритетными строками с корректными состояниями и переходами; посещаемость вынесена в отдельный нейтральный сегмент. Счётчик вкладки теперь суммирует все реальные задачи.
