# Операционная память хранится рядом с кодом

## Контекст

Контекст FitCRM был распределён между корневыми документами, чатами и персональным Obsidian Vault. Это мешает продолжать работу на другом компьютере и приводит к устаревшим копиям.

## Решение

Канонический проектный Vault хранится в `/knowledge` и версионируется вместе с кодом. Локальные `.obsidian` settings игнорируются. Автоматизация меняет только AUTO-секции, а крупные корневые документы не удаляются без отдельной миграции.

## Последствия

Любой компьютер после clone/pull получает одинаковую операционную память. Vault остаётся читаемым без плагинов. Значимые задачи обязаны завершаться docs sync и validation.

## Риски

Документация может создавать шум в Git; поэтому Daily не создаются без работы, changelog отбирается вручную, а task-файл нужен только для содержательной задачи.
