# Google Calendar синхронизируется только по явному действию сотрудника

## Контекст

FitCRM подключает рабочий Google Calendar клуба и показывает его события внутри CRM. Посещения
содержат персональные данные клиента и не должны автоматически уходить во внешний сервис сразу
после OAuth.

## Проблема

Полная автоматическая синхронизация создаёт неожиданные события, раскрывает больше данных, чем
сотрудник собирался передать, и усложняет исправление ошибок и дублей.

## Рассмотренные варианты

1. Автоматически переносить расписание и посещения после подключения.
2. Синхронизировать всё по общей кнопке.
3. Показывать Google Calendar в CRM, а создание событий и перенос посещений выполнять только по
   явному выбору сотрудника.

## Решение

Принят вариант 3. OAuth только подключает аккаунт. Сотрудник отдельно создаёт событие или отмечает
конкретные посещения и подтверждает перенос. Повторный перенос одного посещения обновляет
детерминированное Google-событие и не создаёт дубль.

## Причины

- Минимизация неожиданной передачи клиентских данных.
- Понятный контроль сотрудника над содержимым календаря.
- Простая идемпотентность и предсказуемое исправление записей.
- Подключение и отключение не меняют пользовательские события без отдельного действия.

## Последствия

- В интерфейсе всегда видны выбранные записи и явная кнопка переноса.
- Отключение отзывает токен, но не удаляет уже созданные сотрудником события.
- Для массового переноса позже потребуется отдельный подтверждаемый сценарий, а не скрытый cron.

## Риски

- Ручной выбор требует больше действий.
- OAuth Calendar scope требует Google verification для снятия user cap и предупреждения.

## Условия пересмотра

Решение можно пересмотреть после обезличенного анализа использования и отдельного UX согласия на
правила автоматизации для каждого типа данных.
