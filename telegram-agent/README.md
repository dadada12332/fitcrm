# FitCRM Telegram Agent 🤖

Оркестратор команды агентов FitCRM через Telegram. Ты пишешь задачу в чат — бот
запускает **Claude Code в headless-режиме** в репозитории, лид разбирает задачу и
подключает ролевых субагентов из [`../.claude/agents/`](../.claude/agents), а прогресс
и результат прилетают обратно в чат.

```
ТГ-сообщение (задача)
   → [этот воркер] → claude -p "…" --output-format stream-json  (в папке fitcrm)
   → лид вызывает 🧪 qa-tester / 🔒 security-auditor / 🎨 ui-designer / 🗄️ db-architect / 🔍 code-reviewer
   → шаги и итог постятся в чат
```

## ⚠️ Безопасность (обязательно к прочтению)

Воркер запускает Claude с `--permission-mode bypassPermissions` — он **может читать и
править код, выполнять команды и деплоить** без подтверждений. Это фактически удалённое
выполнение кода, инициируемое из Telegram. Поэтому:

- **Обязательно** заполни `TELEGRAM_ALLOWED_IDS` — только твой user_id (и/или id вашей группы).
  Пустой список = бот никого не пускает.
- Заведи **отдельного** бота (@BotFather), не переиспользуй боевого клубного бота.
- Хости воркер в приватном окружении. Не давай токен/ключи посторонним.
- Задачи выполняются по очереди (одна за раз), чтобы процессы не мешали правкам в репо.

## Предпосылки

- Установлен и авторизован **Claude Code CLI** (`claude`) — воркер вызывает его как подпроцесс.
  Проверь: `claude --version`. Авторизация — через `ANTHROPIC_API_KEY` или `claude login`.
- Репозиторий `fitcrm` на той же машине (воркер по умолчанию работает в родительской папке).
- Node 18+.

## Запуск локально (для теста)

```bash
cd telegram-agent
cp .env.example .env      # заполни TELEGRAM_AGENT_BOT_TOKEN, TELEGRAM_ALLOWED_IDS, ANTHROPIC_API_KEY
npm install
# подгрузить .env и запустить:
node --env-file=.env worker.mjs
```

Напиши боту `/start`, затем любую задачу текстом. Бот работает на long-polling —
внешний URL/вебхук не нужен.

## Деплой (always-on)

Vercel **не подходит** (serverless, долгие задачи). Нужен always-on воркер:

- **Railway / Fly.io / VPS**: задеплой папку `telegram-agent`, задай переменные окружения,
  команда старта `node --env-file=.env worker.mjs` (или через переменные окружения хостинга
  без `.env`). На хосте так же должен быть установлен и авторизован `claude` CLI и лежать
  репозиторий (или примонтирован), т.к. лид работает прямо в коде.
- Ограничение: одна машина = один рабочий репозиторий. Правки/деплой идут в него.

## Команды бота

- `/start`, `/help` — справка
- `/queue` — сколько задач в очереди
- любой текст — новая задача

## Стоимость

Каждая задача — реальные токены Claude (зависит от объёма работы). Держи это в уме при
формулировке задач.
