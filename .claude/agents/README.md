# Команда агентов FitCRM

Ролевые субагенты для Claude Code. **Лид — основной Claude** (оркестратор): он читает задачу,
декомпозирует, вызывает нужного агента (или нескольких), сводит результаты и общается с пользователем.
Субагенты запускаются в рамках сессии, делают работу и возвращают лиду отчёт — они **не** автономные
процессы и не «живут» сами по себе между сессиями.

## Роли

| Агент | Роль | Правит код? | Модель |
|-------|------|-------------|--------|
| `qa-tester` | Функциональное QA: трассировка флоу, сборка, безопасные smoke-тесты, чек-листы | ❌ только отчёт | sonnet |
| `security-auditor` | Пентест по коду: RLS, права в Server Actions, инъекции, утечки | ❌ только отчёт | opus |
| `ui-designer` | UI/UX: лендинг, страницы CRM, компоненты, ревью визуала, i18n | ✅ | sonnet |
| `db-architect` | Схема, миграции, RLS, RPC, индексы | ✅ | sonnet |
| `code-reviewer` | Ревью изменений перед коммитом | ❌ только отчёт | sonnet |

## Как лид их вызывает
- «Проверь весь сценарий регистрации» → `qa-tester`
- «Проверь безопасность / нет ли дыр» → `security-auditor`
- «Переделай блок X красиво / добавь страницу» → `ui-designer`
- «Нужна миграция / поправить RLS» → `db-architect`
- «Отревьюй перед коммитом» → `code-reviewer`

Read-only агенты (`qa-tester`, `security-auditor`, `code-reviewer`) находят и репортят; правки делает лид
или профильный агент (`ui-designer`/`db-architect`). Все следуют `../../CLAUDE.md`.
