## 3. Фаза Production

**Общий статус: 🚧 ~60% — блокеры перед первым платящим клиентом**

| Функция | Статус | Приоритет | Комментарий |
|---|---|---|---|
| Приём онлайн-платежей (Payme/Click/Uzum) | ❌ | 🔴 | Сейчас только enum-лейблы; нет интеграции шлюзов |
| Шифрование PII (имена, телефоны, зарплаты) | ❌ | 🔴 | В схеме помечено «шифрование позже», не сделано |
| Резервное копирование / restore | 📋 | 🔴 | Полагается на автобэкапы Supabase; своей стратегии нет |
| Автопродление подписок SaaS | 🚧 | 🟠 | Есть заявки (`billing_requests`), автосписания нет |
| Audit-трейл в UI | 🚧 | 🟠 | Таблица `audit_logs` есть; заполнение/просмотр частичны |
| Мониторинг/алерты (Sentry) | ❌ | 🟠 | Не подключены |
| Rate limiting / anti-abuse | ❌ | 🟠 | Отсутствует |
| Реальный SMS-провайдер | ❌ | 🟡 | OTP идёт через Telegram |
| Уведомления клиентам (истечение/долг) | 🚧 | 🟠 | Инфраструктура `notifications` есть, автосценарии частичны |
| Полное покрытие RLS-тестами | 📋 | 🟠 | Проверялось вручную |
| Тарифные лимиты по планам (enforcement) | 🚧 | 🟠 | Планы есть, жёсткого ограничения функций по плану нет |

## 6. Сводная таблица

| Фаза | Готовность | Ключевой блокер для перехода дальше |
|---|---|---|
| **MVP** | ✅ ~95% | Доделать classes/bookings UI, базу знаний |

**Рекомендованный ближайший фокус (до первого платящего клуба):**
1. 🔴 Интеграция Payme/Click (приём оплат).
2. 🔴 Шифрование PII + аудит доступа.
3. 🔴 Стратегия бэкапов и восстановления.
4. 🟠 Sentry + базовый мониторинг ошибок.
5. 🟠 Автосписание/enforcement тарифных лимитов.
